# Codex Sessions Viewer

Codex Sessions Viewer is now split into two runtime pieces:

- `apps/web`: a generic hosted site shell
- `apps/api`: a machine-local bridge daemon that scans `~/.codex`, stores local metadata, serves localhost APIs, exports data, and launches local resume commands

The hosted site never needs raw session data stored remotely in this phase. It probes the local bridge on `127.0.0.1:4318` first and falls back to `127.0.0.1:4319`.

## Workspace

- `packages/core`: shared bridge config, DTOs, and normalization helpers
- `apps/api`: local bridge daemon and localhost API
- `apps/web`: hosted-web shell and local-bridge client

- override hosted site URL with `CSV_HOSTED_SITE_URL`

## GitHub Pages

The hosted shell is built from `apps/web` and published with GitHub Pages.

Local build that matches Pages subpath mode:

```bash
CSV_WEB_BASE_PATH=/codex_sessions_viewer/ pnpm --filter @csv/web build
```

GitHub Actions workflow:

- `.github/workflows/pages.yml`
- publishes `apps/web/dist`
- injects `CSV_WEB_BASE_PATH=/${repo}/`

Bridge package workflow:

- `.github/workflows/release-bridge.yml`
- builds the macOS bridge package on every `main` push
- publishes a stable download asset under tag `bridge-latest`

## New Mac Install Flow

The tarball generated by `pnpm package:mac` is the install artifact for another Mac.
The hosted install page links to the rolling GitHub Release asset directly.

1. Download [codex-sessions-viewer-bridge-macos.tar.gz](https://github.com/sivagao/codex_sessions_viewer/releases/download/bridge-latest/codex-sessions-viewer-bridge-macos.tar.gz).
2. Extract it.
3. Run `./install.sh`.
4. The installer bootstraps the bridge and opens the hosted viewer automatically.
5. If you need manual commands later, use the quoted absolute paths under `~/Library/Application Support/CodexSessionsViewerBridge/bin/`.

The installer places the bridge under:

- `~/Library/Application Support/CodexSessionsViewerBridge`

It also installs a `launchd` agent so the local bridge daemon can run in the background after login, and creates helper symlinks under `~/.local/bin/`.

## Another Mac From Source

```bash
git clone <your-repo-url> codex_sessions_viewer
cd codex_sessions_viewer
pnpm install
pnpm bridge:open
```

`bridge:open` starts the local bridge if needed and opens the hosted site URL in the default browser.

Current phase guarantees:

- each machine reads its own `~/.codex`
- favorites/tags/notes stay local
- raw sessions never leave the machine

## launchd

Use the template at `scripts/macos/codex-sessions-viewer.plist.template` if you want the local bridge daemon to run in the background on login.
